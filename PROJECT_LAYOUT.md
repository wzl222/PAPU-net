# Project Layout

This file summarizes the current top-level layout of the PromptIR workspace.

## Core code

- `net/`: model definitions
- `utils/`: helper functions and losses
- `train.py`, `train_uieb.py`: training entry points
- `infer_water_promptir.py`: inference entry point
- `evaluate_new.py`: evaluation script
- `uiqm_postprocess.py`: post-processing utilities
- `run_uiesam_infer.py`, `run_uiqm_search.py`: experiment scripts

## Data and checkpoints

- `data/`, `data_dir/`: dataset-related files and splits
- `ckpt/`: original pretrained checkpoints
- `train_ckpt/`: training outputs and saved checkpoints

## Outputs

- `output/`: restored images generated by inference
- `logs/`: training, inference, and search logs
- `result/`: selected qualitative example images
- `reports/`: organized metrics and paper figures
- `analysis/`: feature-analysis experiments and t-SNE results

## Documents

- `README.md`, `INSTALL.md`, `LICENSE.md`: project docs
- `docs/paper_assets/`: paper-related notes and draft files

## Notes

- Experimental metric files have been moved from the project root into
  `reports/metrics/`.
- Loose top-level figures used for reporting have been moved into
  `reports/figures/`.
